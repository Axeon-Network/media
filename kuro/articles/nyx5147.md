**Nyxeon build 5147** is a build of [Nyxeon](Nyxeon). It was publicly released on May 1st 2026 on the URL [axeon-network.github.io/nyxeon](https://axeon-network.github.io/nyxeon). This build is the second publicly available build of Nyxeon, the first being [build 4732](Nyxeon_build_4732) (which was released as a source packet hosted on catbox.moe instead of being publicly deployed to the [Axeon Network](Axeon_Network) website). 

Based on the codebase of [KuroWiki](KuroWiki) Release Candidate 1, it has served as a test-in-production project by the Axeon Network in order to test core-changing features anywhere else than development builds of KuroWiki, in order to avoid breaking the site and losing progress in case said features fail.

# Notable features
This build is notable for not using the Material Design Lite-based layout present in other Deltari-based projects, such as *[Deltari Meta](Deltari_Meta)*, using a simple custom from-scratch layout (codenamed *Nyx-CM*) instead. Most parts of this build reports itself as "Axeon Deltari Code Name "Cairo"", named after the capital of [Egypt](https://en.wikipedia.org/wiki/Egypt), which is the code name for the first planned public release of the self-hostable edition of Deltari, sometimes also called *DeltariSH*.

Alongside that, the Internal Componentization Method has been introduced, which is an internal project that separates the aricles, images and the core of Deltari itself in order to prevent another [KuroWiki Beta 6 situation](Development_of_Deltari#dummy) <!-- i gotta change the url later. -->

The articles are hosted on a [public repo](github.com/Axeon-Network/db-nyxn) dubbed the "Nyxeon Database", which hosts the raw Markdown files for each article, then the Nyxeon repository itself fetches the database via Git Submodules pointed to `/pages/articles`. The `mediaurl` site variable was added to set the URL where images are hosted.

# Bugs and quirks
- Debug Mode is enabled by default, thus the site and SpringViewer watermarks are shown to the end-user. As a result of this, the Deltari Test Page is shown on the navigation bar.
- All and any instances of the site's version number have become blank, which is caused by [Panther](Panther) not running under GitHub Pages since GitHub Pages does not allow custom plugins to run, normally dubbed as running in safe mode. The build tag can be seen at `/res/ruby/buildtag` on the source code, since that is where Panther writes it.
- Any component that relies on MDL doesn't work, the most notable case of this is SpringViewer, where most assets use their respective HTML fallbacks.
- Light Mode does not work in this build and has been dummied out in the source code. Additionally, its respective checkbox in the Settings page has been disabled, showing a message to the user that the layout is not compatible with the latter.
- The homepage description has the raw \`config.yml` string, since it does not allow the use of Markdown.
